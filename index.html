Angular.js

HTML enhanced for web apps!

Created by Andreas Taube / @andreast00

What is Angular.js?

  • A lightweight (meta) framework which teaches the browser new tricks!
  • 100% JavaScript, 100% client side and compatible with both desktop and mobile browsers.
  • Multi-Browser Support:
    Safari, Chrome, Firefox, Opera, IE8, IE9
    Android, Chrome Mobile, iOS Safari.
  • File size < 29KB (compressed)

Benefits

  • Makes development of web apps extremely easy and fast.
  • Clear seperation of application logic, presentation components and service layers.
  • Extremely test friendly.
  • Strong reusability by extending HTML with custom DSL.
  • Rich feature set with very common use cases taken care off.
  • No angular specifiy API calls required -> easy intergration with 3rd party libraries.
  • Suitable for single-page and rount-trip apps / integration projects.
  • Good documentation + strong community support (Google)

Features

  • Dependency Injection
  • Service Abstraction
  • Directives
  • Expressions
  • Forms (validation)
  • i18n
  • Modules
  • Filters
  • REST Client

Two-Way Data Binding

Angular compiles directives into a live view combined with a scope model. The result is a continuously updating view which does not need template model re-merging.

No action from the developer is needed to update the view.

The code can get hold of the elements and register event handlers and know that the reference will not be destroyed by template data merge.

Expressions and Filters

Expressions

Expressions are JavaScript-like code snippets that are usually placed in bindings such as {{ expression }}. They are mostly used in views as HTML markup.

							
1+2

"text".toUpperCase()

myModel.myProperty

Filters

  • Used to convert raw data into a user-friendly (readable) format.
  • Can be chained by pipes like in bash.
  • May accept arguments.

							
value | filter1 | filter2

123 | number:2
  • Available filters:
    currency, date, filter, json, limitTo, lowercase, uppercase, number, orderBy

View / Scope / Controller

View / Scope / Controller

  • View: HTML (declarative)
  • Scope: Glue between view and controller.
  • Controller: JavaScript (imperative)
    No DOM manipulation, no Angular specific api calls.

Scope hierarchies

  • Scopes are arranged in hierarchical structure and mimic the DOM strucutre of the application.
  • Each Angular application has exactly one root scope, but may have several child scopes.

Directives

Directives

  • Extend the HTML vocabulary in a declarative way.
  • Attach behavior to any HTML element or attribute.
  • Create new HTML elements or attributes with custom behavior (DSL).
							
	<span ng-form="exp"></span>
	<span class="ng-form: exp;"></span>
	<ng-form></ng-form>
	<!-- directive: ng-form exp -->

Angular provides a set of pre defined directives which are usefull for building web apps. They all use the ng-prefix.

A directive is just a function which executes when the compiler encounters it in the DOM.

Summary

Pros

  • Suitable for small and big development projects.
  • Bidirectional data binding.
  • Clear seperation of concerncs -> easy testability!
  • HTML can be extended with custom DSL -> great for reuseability!
  • No dependecy on 3rd party libraries.
  • Add-on projects i.e. angular-ui, angular-bootstrap-ui, etc...

Cons

  • Many new concepts which are hard to understand (in the fist moment) coming from jQuery like app development.
  • Steep learning curve.
  • Complex data bindings might cause performance issues (>1000 bindings on one page)
